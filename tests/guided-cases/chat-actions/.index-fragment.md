# Chat Actions — Guided Test Cases

## Flow: chat-actions
Action proposals, confirmation workflow, and edge cases for the AI Chat action system.

| ID | Title | Type | Priority | File |
|----|-------|------|----------|------|
| GC-009 | Request Action and Confirm | positive | P0 | `gc-009-confirm-action.md` |
| GC-010 | Action Proposal Display Format | positive | P1 | `gc-010-action-display.md` |
| GC-011 | Request Action Without Confirmable Emails | edge | P1 | `gc-011-no-matching-action.md` |

## Coverage
- **Positive flow**: Full action lifecycle — request, proposal, confirm, success message (GC-009)
- **UI verification**: Action proposal rendering — description text, Confirm button styling, ACTION_PROPOSAL stripping (GC-010)
- **Edge case**: No matching emails for requested action — graceful degradation (GC-011)
