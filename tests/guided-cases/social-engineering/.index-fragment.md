## Coverage Summary
- **Happy path (POST detect)**: suspicious email returns high/critical risk with tactics (GC-349), clean email returns none (GC-350)
- **Short-body fast path**: body < 20 chars bypasses AI and returns none immediately (GC-351)
- **Caching — POST**: repeat POST returns identical cached result (GC-352)
- **Caching — GET**: GET endpoint returns cached result after POST analysis (GC-353)
- **Negative — GET before POST**: GET on unanalyzed message returns 404 (GC-354)
- **Negative — unknown message**: POST with non-existent message_id returns 404/400 (GC-355)
- **Validation — tactic type allowlist**: unrecognized tactic types from AI are filtered out (GC-356)
- **Auth guard**: both POST and GET endpoints reject unauthenticated requests with 401 (GC-357)
- **Validation — risk/tactic invariant**: risk_level > low with empty tactics is downgraded to "none" (GC-358)
- **API endpoints exercised**: POST /api/ai/detect-social-engineering, GET /api/messages/{id}/social-engineering
- **Tactic types covered**: urgency_pressure, fear_threat (GC-349); full allowlist validation (GC-356)
- **Risk levels covered**: none (GC-350, GC-351, GC-358 downgraded), medium/high/critical (GC-349)
