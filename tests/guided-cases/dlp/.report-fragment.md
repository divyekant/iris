### Failure Details

**GC-360 — Mask format** (`mask_value` in `src/api/dlp.rs`):
- Implementation: `first4 + "*".repeat(len - 8) + last4` → for 16-digit card: 8 stars
- Case spec expected: `4111****1111` (4 stars)
- Fix: either update spec to match implementation (8 stars is technically more conservative masking), or change `mask_value` to always use exactly 4 stars for card numbers

**GC-361 — SSN sequential filter**:
- `is_sequential()` in `src/api/dlp.rs` only checks `s == "123456789" || s == "987654321"`
- `123-45-6789` → digits `123456789` matches the first branch → filtered as sequential
- Case note says this value should "pass through to detection" but implementation treats it as a known false-positive pattern
- Fix: either remove 123456789 from sequential exclusions (it is a valid-format SSN per spec), or update the case to use a different SSN like `219-09-9999`

**GC-362 — API key risk level**:
- `determine_risk_level()` classifies `FindingType::ApiKey` as High unconditionally
- Spec says single api_key = Low, multiple findings = escalate to High
- Fix: change `determine_risk_level` to return `Low` for a single api_key or password finding, `High` only when there are 2+ findings or the finding type is credit_card / ssn / private_key
