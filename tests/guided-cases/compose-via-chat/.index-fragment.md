## Coverage
- **Happy path (P0)**: Full compose lifecycle — chat request → `COMPOSE_PROPOSAL` parsed → `propose_action` stored → confirm → draft saved in `Drafts` folder (GC-339)
- **Decline flow (P0)**: Not calling confirm leaves zero drafts created; the draft lifecycle requires explicit user confirmation (GC-340)
- **CC recipients (P1)**: `cc` array in `ComposeEmailArgs` is populated and persisted in `cc_addresses` (GC-341)
- **Reply with context (P1)**: `reply_to_message_id` wires up thread context lookup; `thread_subject` is populated from the DB at compose time (GC-342)
- **Missing `to` validation (P1)**: `handle_compose_email` rejects empty `to` array; no draft persisted (GC-343)
- **Missing `subject` validation (P1)**: `handle_compose_email` rejects blank subject; no draft persisted (GC-344)
- **Invalid confirm (P1)**: Non-existent `message_id` → 404; wrong session → 404; user-role message (no proposed_action) → 400 (GC-345)
- **Field verification (P0)**: All 10 draft fields checked: `is_draft`, `folder`, `to_addresses`, `subject`, `body_html`, `body_text`, `snippet`, `is_read`, `account_id`, `date` (GC-346)
- **Auth enforcement (P0)**: Both `/api/ai/chat` and `/api/ai/chat/confirm` require `X-Session-Token`; missing or forged token → 401 (GC-347)
- **XSS storage behavior (P1)**: XSS in compose body is stored as-is (API does not render it); JSON encoding escapes angle brackets; `body_text` has tags stripped by `strip_html_tags` (GC-348)

## Key Code Paths
- `src/api/chat.rs` — `chat()`, `confirm_action()`, `parse_action_proposal()` (COMPOSE_PROPOSAL parsing)
- `src/ai/tools.rs` — `compose_email` tool definition, `handle_compose_email()` (validation), `execute_compose_email()` (draft INSERT), `ComposeEmailArgs`, `ComposeEmailData`
- Proposal flow: `COMPOSE_PROPOSAL:{...}` in AI text → `parse_action_proposal` → `ProposedAction { action: "compose_email", data: ComposeEmailArgs }` stored in `chat_messages.proposed_action`
- Confirm flow: `proposed_action.action == "compose_email"` branch in `confirm_action` → `execute_compose_email` → `INSERT INTO messages (..., is_draft=1, folder='Drafts')`
