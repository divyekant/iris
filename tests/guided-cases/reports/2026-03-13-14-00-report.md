## Bugs Found and Fixed During This Session

### BUG-1: confirm_action returns HTTP 500 (src/api/chat.rs)
**Root cause 1 (params_from_iter):** The `confirm_action` handler built params as `Vec<Box<dyn rusqlite::types::ToSql>>`. When passed to `rusqlite::params_from_iter`, the `&dyn ToSql` reference does not satisfy rusqlite's `ToSql` bound, causing the execute to fail.
**Fix 1:** Changed to `Vec<rusqlite::types::Value>` (concrete enum that implements `ToSql` directly).

**Root cause 2 (UNIQUE constraint):** The `messages` table has a `UNIQUE(account_id, message_id, folder)` index (migration 006). Updating `folder = 'Archive'` on messages that already have an Archive row (IMAP multi-folder) caused constraint violations.
**Fix 2:** Changed `UPDATE messages SET folder = 'Archive'` to `UPDATE OR IGNORE messages SET folder = 'Archive'` (and same for trash).

**Docker cache issue:** Docker's BuildKit was caching the `RUN cargo build --release` layer even after source changes. Required `--no-cache` rebuild to get the fix into the container binary.

## Notes and Observations

1. **API response shape mismatches (non-blocking):**
   - `POST /api/ai/scan-followups` returns `reminders_created` key, spec documents `created`
   - `GET /api/followups` returns `{"reminders": [...], "total": N}`, spec expects plain array
   - `PUT /api/followups/{id}/snooze|dismiss|acted` returns 204 No Content, spec expects 200 with updated object

2. **Bulk chat action AI behavior:** The AI sometimes executes bulk actions via `bulk_update_emails` tool directly instead of producing `ACTION_PROPOSAL:{json}` text. Prompting explicitly to "propose the action first with ACTION_PROPOSAL format" consistently triggers the proposal-then-confirm flow.

3. **Docker BuildKit caching:** The `RUN cargo build --release` layer is cached by content hash. Source edits invalidate the COPY layer but the Rust build layer may persist across rebuilds. Always use `--no-cache` when fixing Rust source bugs and deploying.

4. **move_to_category description parsing:** The `rsplit(" to ")` extraction picks up trailing words like "category" from descriptions like "Move emails to the promotions category", resulting in `ai_category = "the promotions category"` instead of `"promotions"`. Functional but stores verbose category name.

5. **Followup reminder deduplication:** The `GET /api/followups` response sometimes returns duplicate rows (same `id` appearing twice). This is a JOIN artifact from the `LEFT JOIN messages` when the message_id matches multiple message rows. The unique IDs count is correct.
