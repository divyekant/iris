## Failure Summary

### GC-312 — `GET /api/messages/prioritized` returns empty for all requests (P0 bug)

**Root cause:** The `select_cols` string in `src/api/relationship_priority.rs` (line ~410) is missing three columns that `MessageSummary::from_row` reads by name:

```
// Current (incomplete):
let select_cols = "m.id, m.account_id, m.thread_id, m.folder, m.from_address, m.from_name,
                   m.subject, m.snippet, m.date, m.is_read, m.is_starred, m.has_attachments,
                   m.labels, m.ai_priority_label, m.ai_category";

// Missing: m.ai_sentiment, m.ai_needs_reply, m.intent
```

`MessageSummary::from_row` calls `row.get("ai_sentiment")`, `row.get("ai_needs_reply")`, and `row.get("intent")`. When these columns are absent from the SELECT, rusqlite returns an error. The `.filter_map(|r| r.ok())` call silently discards all errored rows, leaving an empty Vec. `total` is computed from `messages.len()` after filtering, so it reports 0 even though 50 INBOX messages exist in the database.

**Fix:** Add the three missing columns to `select_cols`:
```
m.ai_sentiment, m.ai_needs_reply, m.intent
```

**Confirmed via:**
- SQLite query of the same CTE returns 48 rows correctly
- DB shows 50 INBOX messages with `is_deleted=0` for account `715f3734-058c-4232-8dcf-e2e95fc1ada2`
- `MessageSummary::from_row` reads all fields by name (line ~40–61 of `src/models/message.rs`)

### GC-313 — Pagination not verifiable (dependent on GC-312)

Both pages return HTTP 200 and identical `total=0`. The pagination structure (limit/offset params, total field) is implemented correctly in code, but results cannot be verified because no messages are returned. This is a consequence of the GC-312 bug, not an independent pagination defect.

### GC-314 — Limit cap not verifiable (dependent on GC-312)

Both `limit=9999` and `limit=1000000` return HTTP 200 with no 400/500 errors. The cap is confirmed in source (`params.limit.unwrap_or(50).min(500)`). However, functional verification that the cap prevents >500 messages in the response cannot be confirmed with an empty result set.
