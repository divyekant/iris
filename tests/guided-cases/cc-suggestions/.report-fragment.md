### Notes

**Infrastructure discovery:** `localhost` on this machine resolves to `::1` (IPv6), which is intercepted by OrbStack before reaching iris-server. All tests were executed against `127.0.0.1:3000` (IPv4) to reach iris-server directly.

**AI availability:** Anthropic provider is healthy and AI is enabled. However, GC-319/320/325/326 are SKIPped because the live DB has no co-occurrence data — all synced messages are single-sender threads (StockX, mailer-daemon) with no CC addresses. The co-occurrence query returns empty, so the handler short-circuits before calling AI. This is correct behavior; the precondition "known sender with co-occurrence history" is not met.

**GC-322 partial:** The deduplication logic is exercised and verified via unit tests (`test_co_occurrence_excludes_recipients`). The HTTP test confirms no suggestions are returned when CC'd contacts are present, but the full "other contacts may still appear" criterion requires seed data.

**Unit test coverage:** All 17 unit tests in `cc_suggestions.rs` pass, covering co-occurrence queries, thread CC extraction, prompt building, JSON parsing, confidence clamping, type normalization, and max-5 truncation.
