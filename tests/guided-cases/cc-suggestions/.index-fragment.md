## Coverage Summary
- **Happy path**: co-occurrence pipeline end-to-end with known recipients (GC-319), thread history enriching suggestions via `thread_id` (GC-320)
- **Response contract**: confidence in [0.0, 1.0] ordered descending (GC-325), max 5 suggestions enforced regardless of candidate pool size (GC-326)
- **Input validation**: empty `to` returns empty or 400 (GC-321), already-CC'd contacts excluded from suggestions (GC-322), empty body preview accepted (GC-324)
- **No-data edge cases**: unknown recipient with no co-occurrence history returns empty suggestions without hallucinating (GC-323)
- **Input length**: over-length `body_preview` handled gracefully (GC-324)
- **Security**: unauthenticated requests (no token, forged token, empty token) return 401 (GC-327), XSS payloads in `subject` and `body_preview` do not execute or appear unescaped in response (GC-328)
- **API**: POST /api/ai/suggest-cc (all 10 cases)
