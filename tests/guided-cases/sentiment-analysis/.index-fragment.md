## Key Implementation Details
- `ai_sentiment` column added via `migrations/019_sentiment.sql` (`ALTER TABLE messages ADD COLUMN ai_sentiment TEXT`)
- Column is nullable — messages processed before the feature or when no AI provider is available will have `NULL`
- Valid enum values: `positive`, `negative`, `neutral`, `mixed` — defined in the `SYSTEM_PROMPT` in `src/ai/pipeline.rs`
- `MessageSummary` and `MessageDetail` both expose `ai_sentiment: Option<String>` (`src/models/message.rs`)
- The list endpoint selects `ai_sentiment` in the `select_cols` string (`src/api/messages.rs`)
- UI pill in `web/src/components/inbox/MessageRow.svelte`: guarded by `sentimentConfig[message.ai_sentiment]` lookup — unknown/invalid values are silently hidden
- Color mapping: `positive` → `--iris-color-success` (green), `negative` → `--iris-color-error` (red), `neutral` → `--iris-color-text-faint` (gray), `mixed` → `--iris-color-warning` (yellow)
- Known accessibility gap: pill uses `title` attribute only; `aria-label` is missing (GC-148)
- No sentiment filter param on `GET /api/messages` — `ListMessagesParams` only accepts `account_id`, `folder`, `category`, `limit`, `offset` (GC-147)
