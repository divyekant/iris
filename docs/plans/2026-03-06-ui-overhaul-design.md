# Iris UI Overhaul — Design Document

**Date:** 2026-03-06
**Status:** Approved
**Prototypes:** `docs/designs/iris-ui-overhaul.pen` (14 screens)

## Decision: Option C — Top Nav Layout

Full-width horizontal top navigation bar with no persistent sidebar. Maximizes content width, scales via overflow menu, and keeps all screens consistent.

### Why Option C

- **Maximum content width** — no 224px sidebar eating horizontal space
- **Scalable navigation** — primary items visible, overflow dropdown for extras (Archive, Starred, Trash)
- **Consistent pattern** — every page uses the same top nav, no layout shifts
- **Chat panel friendly** — AI Chat slides in from right without competing with a sidebar

---

## Skinnable Architecture

### Design Token System

All visual properties flow through CSS custom properties generated by Kalos (`.kalos.yaml`). Structure is separated from skin — components reference tokens, never hardcoded values.

**Token categories:**
- Colors: primary, secondary, semantic (success/warning/error/info), backgrounds (3 tiers), text (3 tiers), borders (3 types)
- Typography: font family, mono font, type scale (1.25 ratio, 16px base)
- Spacing: 4px base unit, scale [0,1,2,3,4,6,8,12,16,24,32]
- Radii: none/sm/md/lg/full
- Transitions: fast (120ms), normal (200ms)

**Brand palettes:** Dark (default) and Light, switchable via `data-brand` attribute on `<html>`. Each palette overrides color + typography tokens. Spacing and radii are structural — never overridden by brands.

**Token implementation:** `kalos-tokens.css` generates `:root` defaults + `[data-brand="light"]` overrides. Tailwind config references `var(--kalos-*)` values. Components use Tailwind classes or CSS vars directly — never raw hex values.

### Future skinning

Users can provide their own `.kalos.yaml` with custom palettes. The architecture supports this without touching component code — only token values change.

### Top Nav Bar (56px height)
- **Logo:** "Iris" in primary color, 20px bold
- **Account switcher:** Avatar + email + chevron dropdown (between logo and nav)
- **Nav items:** Icon + label pills, active = gold tint background + gold text
- **Primary visible:** Inbox, Search, Drafts, Sent
- **Overflow "More":** Archive, Starred, Trash, Settings (flyout menu)
- **Right actions:** Sync status pill, AI Chat toggle, Compose button

### Account Switcher Dropdown
- Lists connected accounts with avatar, email, provider, unread count
- Active account has check mark
- "All accounts" option for unified view
- "Add account" at bottom with divider

## Screens Designed

### Core Screens

**Inbox (C3 base):** Top nav → Inbox header (title + badge + category tabs: All/Primary/Updates) → Message list (72px rows: unread dot, sender, subject, preview, timestamp, priority dot)

**Thread View (C2):** Back button → Subject → Collapsible AI Summary panel (sparkles icon) → Message cards (avatar, sender, trust badge, timestamp, body) → Collapsed message previews → Reply/Reply All/Forward action bar

**Compose Modal (C4):** Dark overlay → Centered 680×560 modal → Header ("New Message" + close X) → To/Cc/Subject fields → Body textarea → Footer (Send button gold, AI Assist dropdown with sparkles, Attach button, ⌘Enter hint)

**Search (C5):** Full-width search bar (search icon, query text, clear X) → Filter chips (pill-shaped: Has attachment, Last 7/30/365 days, active = gold tint) → Result count → Result cards (elevated bg, sender, priority, timestamp, subject, snippet)

**Settings (C6):** Top nav → Horizontal split: left settings nav (220px, icon + label rows) → right content panel. Sections: Accounts (card with avatar, email, provider, sync time, status badge, Add Account button), Appearance (Dark/Light/System toggle cards with preview squares), AI Processing, API Keys, Audit Log.

### AI Features

**Inbox + AI Chat Panel (C3):** Content splits horizontally — inbox list (flex) + chat panel (380px fixed, right border). Chat panel: header (sparkles + "AI Assistant" + close X), conversation bubbles (AI = surface bg, User = gold tint, right-aligned), citation cards (mail icon + source), suggestion chips (bordered pills), input bar (text field + gold send button).

### Interactive States

**Bulk Selection (C11):** Checkboxes appear on message rows. Selected = gold filled checkbox + gold tint row background. Gold action bar slides in below nav: "3 selected" count | Archive | Mark Read | Star | Delete (red) | spacer | Clear.

**Account Switcher (C7):** Dropdown from nav button. Rounded card with account rows, check on active, "All accounts", divider, "Add account".

**Nav Overflow (C8):** "More" button with ellipsis icon. Dropdown: Archive, Starred, Trash, divider, Settings.

### Empty & Error States

**Empty Inbox (C9):** Centered vertically — circular icon frame (inbox icon, faint) → "Inbox zero" title → "You've read everything. Time to go outside." subtitle.

**No Search Results (C10):** Search bar with query → centered "search-x" icon → "No results found" → "Try different keywords or remove some filters."

**Loading Skeleton (C13):** "Syncing..." gold pill in nav → skeleton rows (varying-width gray bars mimicking sender/subject/preview layout, 4 rows visible).

**Error/Disconnected (C14):** Red "Sync failed" pill in nav → full-width red error banner (triangle-alert icon, message about expired OAuth, "Reconnect" red button, dismiss X) → dimmed (50% opacity) stale inbox content below.

### Theme Validation

**Light Theme (C12):** Full inbox in light palette — cream background (#FAF9F6), ivory nav (#FFFEF9), warm gold primary (#b8960f), dark brown text (#2C2418), warm muted text (#7A7060), parchment borders (#E8E2D6). Validates the gold brand translates cleanly to light mode.

## What's NOT in scope (future work)

- **Traditional vs Messaging view mode:** Toggle between traditional email list and conversational chat-style thread rendering — needs proper design for the messaging layout and UX
- **Responsive/mobile:** Top nav collapses to hamburger at narrow viewports — implementation detail, not designed here
- **Compact density mode:** Reduce row height from 72px to 48px via CSS variable — trivial toggle
- **Drafts page:** Same layout as inbox, filtered to drafts — no unique design needed
- **Keyboard shortcuts:** Behavior, not visual design
- **Animations:** CSS transitions defined in tokens (120ms/200ms ease), applied during build
