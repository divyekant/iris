# V8: AI Chat Panel Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a conversational AI chat panel where users can query their inbox in natural language, get briefings with email citations, and propose bulk actions with confirmation.

**Architecture:** A sliding chat panel (right sidebar) with conversation history. Backend stores chat sessions in SQLite, uses FTS5 to retrieve relevant emails as RAG context, and calls Ollama for conversational responses. Actions proposed by AI require explicit user confirmation before execution.

**Tech Stack:** Rust/Axum (existing `ai::ollama`, FTS5), Svelte 5, Tailwind CSS 4

**Scoping note:** Newsletter Feed (P6) deferred to V9. Semantic search (embeddings/sqlite-vec) deferred — FTS5 used for context retrieval.

---

## Task 1: Chat Session Backend — Migration + Model

Create chat_messages table and model for storing conversation history.

**Migration:** Add to existing migration or create new table via code:
```sql
CREATE TABLE IF NOT EXISTS chat_messages (
    id TEXT PRIMARY KEY,
    session_id TEXT NOT NULL,
    role TEXT NOT NULL CHECK(role IN ('user', 'assistant')),
    content TEXT NOT NULL,
    citations TEXT, -- JSON array of message IDs referenced
    proposed_action TEXT, -- JSON of proposed action if any
    created_at INTEGER NOT NULL DEFAULT (unixepoch())
);
CREATE INDEX IF NOT EXISTS idx_chat_session ON chat_messages(session_id, created_at);
```
